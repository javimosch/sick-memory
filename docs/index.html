sick-memory 🤮💾

File-based memory system for AI coding agents — centralized storage, git-based scoping, worktree support.

⭐ GitHub ⚡ supercli

⚡ Quick Start

🚀 Also available as a supercli plugin: sc plugins install sick-memory 
sick-memory init
sick-memory remember "Authentication uses JWT with 24h expiry" --type project
sick-memory recall "authentication" --json
sick-memory list --json

✨ Key Features

  • 📦 Centralized storage at ~/.sick-memory/
  • 🌿 Git-based scoping — memory tied to repository root
  • 🔗 Worktree support — all worktrees share memory
  • 🔍 TF-IDF search with word-overlap fallback
  • 📋 JSON output for agent consumption
  • 🤖 Agent bridges for Claude Code, OpenCode, Copilot
  • 💾 Zero infrastructure — files only, no DB

📟 Commands

init

🔧 Initialize memory system

remember

💾 Store a new memory

recall

🔍 Search memories

list

📋 List all memories

edit

✏️ Update a memory

delete

🗑️ Remove a memory

🏷️ Memory Types

👤 userInformation about the person (role, goals, expertise)
💬 feedbackCorrections and confirmations about work approach
📌 projectOngoing work context (who, what, why, when)
🔗 referenceExternal system pointers (URLs, dashboard links)
